---
title: Model base class
description: The Model base class provides change tracking, hydration, serialization, and CRUD operations for synced entities.
---

Every synced entity in Strata Sync extends the `Model` base class. Models are registered with the `ModelRegistry` via decorators and participate in the sync lifecycle automatically.

## Creating a model

Define a model by extending `Model` and applying decorators:

```ts
import { Model, ClientModel, Property, ManyToOne } from "@stratasync/core";

@ClientModel("Task")
class Task extends Model {
  @Property()
  title = "";

  @Property()
  status = "todo";

  @Property()
  priority = 0;

  @ManyToOne("Project", "tasks")
  project!: Project;
}
```

The `@ClientModel` decorator registers the class with the `ModelRegistry` under the name `"Task"`. Each `@Property` decorator registers a synced field that participates in change tracking and delta processing.

## Model properties

| Property      | Type                      | Description                                           |
| ------------- | ------------------------- | ----------------------------------------------------- |
| `id`          | `string`                  | Primary key (UUID). Set automatically on creation.    |
| `hydrated`    | `boolean`                 | Whether lazy references have been resolved.           |
| `store`       | `SyncStore \| undefined`  | Backing store reference, injected by the sync client. |
| `__modelName` | `string`                  | The registered model name (read-only getter).         |
| `__data`      | `Record<string, unknown>` | Raw backing data store for property values.           |

## Instance methods

### `hydrate()`

Resolves all lazy references and collections for this model instance. Returns a `Promise` that resolves to the hydrated model with all references populated.

```ts
// Assumes client is a configured SyncClient instance
const task = await client.get<Task>("Task", taskId);
const hydrated = await task.hydrate();
// hydrated.project is now a resolved Project instance
```

**Signature:** `hydrate(): Promise<Hydrated<this>>`

The `Hydrated<T>` utility type converts all `LazyReference<T>` fields to their resolved types.

### `save()`

Persists pending changes by creating an update transaction. If the model has no `id`, it performs a create instead.

```ts
task.title = "Updated title";
await task.save();
```

**Signature:** `save(): Promise<void>`

### `delete()`

Deletes this model instance through the backing store.

```ts
await task.delete();
```

**Signature:** `delete(): Promise<void>`

### `archive()`

Soft-deletes the model by setting an `archivedAt` timestamp.

```ts
await task.archive();
```

**Signature:** `archive(): Promise<void>`

### `unarchive()`

Restores a soft-deleted model by clearing the `archivedAt` field.

```ts
await task.unarchive();
```

**Signature:** `unarchive(): Promise<void>`

### `changeSnapshot()`

Returns a snapshot of the current pending changes and their original values. Used internally by the transaction system.

```ts
task.title = "New title";
const snapshot = task.changeSnapshot();
// snapshot.changes = { title: "New title" }
// snapshot.original = { title: "Old title" }
```

**Signature:** `changeSnapshot(): ChangeSnapshot`

The `ChangeSnapshot` interface:

```ts
interface ChangeSnapshot {
  changes: Record<string, unknown>;
  original: Record<string, unknown>;
}
```

### `clearChanges()`

Clears all tracked property changes without persisting them.

**Signature:** `clearChanges(): void`

### `toJSON()`

Serializes the model data for persistence or transport.

```ts
const json = task.toJSON();
// { id: "abc-123", title: "My Task", status: "todo", ... }
```

**Signature:** `toJSON(): Record<string, unknown>`

### `_applyUpdate(changes)`

Applies property changes without triggering change tracking. Used internally by the sync engine when applying server deltas.

**Signature:** `_applyUpdate(changes: Record<string, unknown>): void`

## Change tracking

The `Model` base class automatically tracks property changes when decorators are applied. Each decorated property uses a MobX observable box under the hood. When a property value changes, the model records the original value in an internal `modifiedProperties` map.

```ts
const task = await client.get<Task>("Task", taskId);
task.title = "Updated"; // Change is tracked
task.priority = 3; // This change is also tracked

const snapshot = task.changeSnapshot();
// snapshot.changes = { title: "Updated", priority: 3 }
// snapshot.original = { title: "Original", priority: 0 }

await task.save(); // Creates an update transaction from the snapshot
// Changes are cleared after save
```

Change tracking can be suppressed internally (for example, when applying server deltas) using the `_applyUpdate` method, which wraps updates in a tracking suppression scope.

## Type utilities

### `Hydrated<T>`

A utility type that converts all lazy reference fields to their resolved types:

```ts
import type { Hydrated } from "@stratasync/core";

// Before hydration: task.project is LazyReference<Project>
// After hydration: task.project is Project
const hydrated: Hydrated<Task> = await task.hydrate();
```

### `LazyReference<T>`

Represents a reference that may not be loaded yet. Access triggers lazy loading when the sync client is available.

```ts
import type { LazyReference } from "@stratasync/core";

// LazyReference<Project> can be awaited or used with resolvePromise
const project = await resolvePromise(task.project);
```
