title: Event system
description: Subscribe to sync lifecycle events for state management and UI updates.
---

The sync client emits events throughout the sync lifecycle. Subscribe to these events to update UI state, show notifications, or trigger side effects.

## Subscribing to events

Use `onEvent` to subscribe to all event types. It returns an unsubscribe function.

```ts
const unsubscribe = client.onEvent((event) => {
  switch (event.type) {
    case "syncStart":
      console.log("Sync started");
      break;
    case "syncComplete":
      console.log("Synced to ID:", event.lastSyncId);
      break;
    case "syncError":
      console.error("Sync error:", event.error);
      break;
    case "modelChange":
      console.log(`${event.action} ${event.modelName}:${event.modelId}`);
      break;
  }
});

// Clean up when done
unsubscribe();
```

## Specialized subscribers

### onStateChange

Subscribe to sync state transitions only.

```ts
const unsubscribe = client.onStateChange((state) => {
  console.log("New state:", state);
  // "disconnected" | "connecting" | "bootstrapping" | "syncing" | "error"
});
```

### onConnectionStateChange

Subscribe to connection state transitions only.

```ts
const unsubscribe = client.onConnectionStateChange((state) => {
  console.log("Connection:", state);
});
```

## Event types

### syncStart

Emitted when a sync cycle begins.

```ts
{
  type: "syncStart";
}
```

### syncComplete

Emitted when a sync cycle completes successfully.

```ts
{
  type: "syncComplete",
  lastSyncId: number // The highest sync ID received
}
```

### syncError

Emitted when a sync cycle fails.

```ts
{
  type: "syncError",
  error: Error // The error that occurred
}
```

### stateChange

Emitted when the client state transitions.

```ts
{
  type: "stateChange",
  state: SyncClientState
  // "disconnected" | "connecting" | "bootstrapping" | "syncing" | "error"
}
```

### connectionChange

Emitted when the transport connection state changes.

```ts
{
  type: "connectionChange",
  state: ConnectionState
}
```

### outboxChange

Emitted when the pending transaction count changes.

```ts
{
  type: "outboxChange",
  pendingCount: number // Current count of unsynced transactions
}
```

### modelChange

Emitted when a model is inserted, updated, deleted, archived, or unarchived -- either from a local mutation or a server delta.

```ts
{
  type: "modelChange",
  modelName: string,  // such as "Task"
  modelId: string,    // such as "task-123"
  action: ModelChangeAction
  // "insert" | "update" | "delete" | "archive" | "unarchive"
}
```

### rebaseConflict

Emitted when a conflict is detected during transaction rebasing.

```ts
{
  type: "rebaseConflict",
  modelName: string,     // such as "Task"
  modelId: string,       // such as "task-123"
  conflictType: string,  // such as "field_conflict"
  resolution: string     // such as "server-wins"
}
```

## SyncClientState values

| State             | Description                                                       |
| ----------------- | ----------------------------------------------------------------- |
| `"disconnected"`  | Not connected to the server.                                      |
| `"connecting"`    | Establishing a connection.                                        |
| `"bootstrapping"` | Loading initial data from the server.                             |
| `"syncing"`       | Connected and actively syncing deltas. This is the "ready" state. |
| `"error"`         | An unrecoverable error occurred.                                  |

## Usage with React

The React bindings in `@stratasync/react` wrap these events in hooks. You typically don't need to use `onEvent` directly in React applications:

```tsx
import { useConnectionState, usePendingCount } from "@stratasync/react";

function SyncStatus() {
  const { status, lastSyncId, error } = useConnectionState();
  const { count, hasPending } = usePendingCount();

  return (
    <div>
      <span>Status: {status}</span>
      <span>Sync ID: {lastSyncId}</span>
      {hasPending && <span>{count} pending</span>}
      {error && <span>Error: {error.message}</span>}
    </div>
  );
}
```

## Practical example: sync indicator

```ts
const client = createSyncClient({
  /* ... */
});

let pendingCount = 0;
let isOnline = true;

client.onEvent((event) => {
  switch (event.type) {
    case "outboxChange":
      pendingCount = event.pendingCount;
      updateUI();
      break;
    case "connectionChange":
      isOnline = event.state !== "disconnected";
      updateUI();
      break;
    case "syncError":
      showNotification(`Sync failed: ${event.error.message}`);
      break;
  }
});

// Your application's UI helper functions
function updateUI() {
  if (!isOnline) {
    showBadge("Offline");
  } else if (pendingCount > 0) {
    showBadge(`${pendingCount} pending`);
  } else {
    showBadge("Synced");
  }
}
```
