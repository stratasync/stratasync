---
title: Transactions
description: Create, serialize, and batch transactions for offline-first mutations in Strata Sync.
---

Transactions are the atomic unit of mutation in Strata Sync. Every local change -- create, update, delete, archive, or unarchive -- becomes a `Transaction` object that flows through the outbox, the transport sends to the server, and the sync protocol confirms.

## Transaction interface

The `Transaction` interface defines the structure of every mutation in the system:

```ts
interface Transaction {
  clientTxId: string; // Unique ID for idempotency
  clientId: string; // Client instance identifier
  modelName: string; // Name of the model being mutated
  modelId: string; // ID of the model instance
  action: TransactionAction; // Type of mutation
  payload: Record<string, unknown>; // New/changed data
  original?: Record<string, unknown>; // Original data (for rollback)
  state: TransactionState; // Current state in the pipeline
  createdAt: number; // Timestamp (ms since epoch)
  retryCount: number; // Number of retry attempts
  lastError?: string; // Last error message
  syncIdNeededForCompletion?: number; // Required sync ID
  batchIndex?: number; // Batch grouping index
}
```

### TransactionAction

| Action    | Code  | Description                           |
| --------- | ----- | ------------------------------------- |
| Insert    | `"I"` | Creates a new model instance.         |
| Update    | `"U"` | Modifies fields on an existing model. |
| Delete    | `"D"` | Permanently removes a model instance. |
| Archive   | `"A"` | Soft-deletes by setting `archivedAt`. |
| Unarchive | `"V"` | Restores a soft-deleted model.        |

### TransactionState

| State            | Description                                                           |
| ---------------- | --------------------------------------------------------------------- |
| `"queued"`       | In the local outbox, not yet sent to the server.                      |
| `"sent"`         | Sent to the server, awaiting confirmation.                            |
| `"awaitingSync"` | Server confirmed, waiting for the sync ID to arrive via delta stream. |
| `"completed"`    | Fully synced.                                                         |
| `"failed"`       | Failed permanently after all retries.                                 |

## Creating transactions

### createTransaction

The general-purpose factory. All other factory functions delegate to this one.

```ts
import { createTransaction } from "@stratasync/core";

const tx = createTransaction({
  clientId: "client-abc",
  modelName: "Task",
  modelId: "task-123",
  action: "U",
  payload: { title: "Updated title" },
  original: { title: "Original title" },
});
```

**Signature:**

```ts
function createTransaction(options: CreateTransactionOptions): Transaction;
```

**CreateTransactionOptions:**

| Field       | Type                      | Required | Description                 |
| ----------- | ------------------------- | -------- | --------------------------- |
| `clientId`  | `string`                  | Yes      | Client instance ID.         |
| `modelName` | `string`                  | Yes      | Registered model name.      |
| `modelId`   | `string`                  | Yes      | Model instance ID.          |
| `action`    | `TransactionAction`       | Yes      | Mutation type.              |
| `payload`   | `Record<string, unknown>` | Yes      | Changed data.               |
| `original`  | `Record<string, unknown>` | No       | Original data for rollback. |

### createInsertTransaction

Creates an `"I"` (insert) transaction for a new model instance.

```ts
import { createInsertTransaction } from "@stratasync/core";

const tx = createInsertTransaction(
  "client-abc", // clientId
  "Task", // modelName
  "task-123", // modelId
  {
    // data (full model)
    title: "New Task",
    status: "todo",
    priority: 0,
    projectId: "project-456",
  }
);
```

### createUpdateTransaction

Creates a `"U"` (update) transaction with changed fields and their original values.

```ts
import { createUpdateTransaction } from "@stratasync/core";

const tx = createUpdateTransaction(
  "client-abc", // clientId
  "Task", // modelName
  "task-123", // modelId
  { title: "Updated title" }, // changes
  { title: "Original title" } // original values
);
```

### createDeleteTransaction

Creates a `"D"` (delete) transaction. Requires the original data for undo support.

```ts
import { createDeleteTransaction } from "@stratasync/core";

const tx = createDeleteTransaction(
  "client-abc", // clientId
  "Task", // modelName
  "task-123", // modelId
  {
    // original data (for undo)
    title: "Task to delete",
    status: "done",
  }
);
```

### createArchiveTransaction

Creates an `"A"` (archive) transaction for soft-deleting a model.

```ts
import { createArchiveTransaction } from "@stratasync/core";

const tx = createArchiveTransaction(
  "client-abc", // clientId
  "Task", // modelName
  "task-123", // modelId
  {
    original: { archivedAt: undefined },
    archivedAt: "2025-01-15T10:30:00.000Z",
  }
);
```

### createUnarchiveTransaction

Creates a `"V"` (unarchive) transaction for restoring a soft-deleted model.

```ts
import { createUnarchiveTransaction } from "@stratasync/core";

const tx = createUnarchiveTransaction(
  "client-abc", // clientId
  "Task", // modelName
  "task-123", // modelId
  {
    original: { archivedAt: "2025-01-15T10:30:00.000Z" },
  }
);
```

### createUndoTransaction

Creates the inverse transaction for any given transaction. Returns `null` if the transaction can't be undone (for example, a delete without original data).

```ts
import { createUndoTransaction } from "@stratasync/core";

const originalTx = createUpdateTransaction(
  "client-abc",
  "Task",
  "task-123",
  { title: "New" },
  { title: "Old" }
);

const undoTx = createUndoTransaction(originalTx);
// undoTx.action === "U"
// undoTx.payload === { title: "Old" }
// undoTx.original === { title: "New" }
```

**Undo mapping:**

| Original action   | Undo action       | Notes                                                         |
| ----------------- | ----------------- | ------------------------------------------------------------- |
| Insert (`"I"`)    | Delete (`"D"`)    | Deletes the inserted model.                                   |
| Delete (`"D"`)    | Insert (`"I"`)    | Re-creates with original data. Returns `null` if no original. |
| Update (`"U"`)    | Update (`"U"`)    | Swaps payload and original. Returns `null` if no original.    |
| Archive (`"A"`)   | Unarchive (`"V"`) | Restores the model.                                           |
| Unarchive (`"V"`) | Archive (`"A"`)   | Re-archives the model.                                        |

## Batching

### createTransactionBatch

Groups multiple transactions into a single batch for atomic server submission.

```ts
import {
  createTransactionBatch,
  createInsertTransaction,
  createUpdateTransaction,
} from "@stratasync/core";

const batch = createTransactionBatch([
  createInsertTransaction("client-abc", "Task", "task-1", { title: "Task 1" }),
  createUpdateTransaction(
    "client-abc",
    "Project",
    "proj-1",
    { taskCount: 5 },
    { taskCount: 4 }
  ),
]);
// batch.batchId: string
// batch.transactions: Transaction[]
// batch.createdAt: number
```

**TransactionBatch interface:**

```ts
interface TransactionBatch {
  batchId: string;
  transactions: Transaction[];
  createdAt: number;
}
```

## Serialization

Transactions can be serialized to a compact format for storage (IndexedDB outbox) or transport (network payloads). The serialized format uses short keys to minimize payload size.

### Single transactions

```ts
import {
  serializeTransaction,
  deserializeTransaction,
  transactionToJson,
  transactionFromJson,
} from "@stratasync/core";

// To compact object
const serialized = serializeTransaction(tx);
const restored = deserializeTransaction(serialized);

// To/from JSON string
const json = transactionToJson(tx);
const parsed = transactionFromJson(json);
```

### Batches

```ts
import {
  serializeBatch,
  deserializeBatch,
  batchToJson,
  batchFromJson,
} from "@stratasync/core";

// To compact object
const serialized = serializeBatch(batch);
const restored = deserializeBatch(serialized);

// To/from JSON string
const json = batchToJson(batch);
const parsed = batchFromJson(json);
```

### Serialized format

The compact format maps field names to short keys:

| Transaction field           | Serialized key |
| --------------------------- | -------------- |
| `clientTxId`                | `cid`          |
| `clientId`                  | `cli`          |
| `modelName`                 | `m`            |
| `modelId`                   | `mid`          |
| `action`                    | `a`            |
| `payload`                   | `p`            |
| `original`                  | `o`            |
| `state`                     | `s`            |
| `createdAt`                 | `t`            |
| `retryCount`                | `r`            |
| `lastError`                 | `e`            |
| `syncIdNeededForCompletion` | `s2`           |
| `batchIndex`                | `b`            |

## Server result types

When transactions are submitted to the server, the response contains result objects:

### TransactionResult

```ts
interface TransactionResult {
  clientTxId: string; // Echoed client transaction ID
  success: boolean; // Whether the transaction succeeded
  syncId?: number; // Assigned sync ID (if successful)
  error?: string; // Error message (if failed)
  errorCode?: string; // Machine-readable error code
}
```

### MutateResult

```ts
interface MutateResult {
  success: boolean; // Whether the batch succeeded
  lastSyncId: number; // Highest sync ID after mutations
  results: TransactionResult[]; // Per-transaction results
}
```
