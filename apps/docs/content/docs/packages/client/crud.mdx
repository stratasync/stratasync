The `SyncClient` provides CRUD operations that work with the identity map, outbox, and optimistic update system.

## Common behavior

Every mutation (`create`, `update`, `delete`, `archive`, `unarchive`) follows the same pattern. It applies the change optimistically to the local identity map, creates a transaction in the outbox, and records a history entry for undo/redo. Each mutation also emits a `modelChange` event so reactive layers (MobX, React) pick up the change immediately.

## Reading models

You can read models synchronously from the identity map or asynchronously from storage and the server.

### get

Fetches a model by ID. Checks the identity map first, then falls back to storage.

```ts
const task = await client.get<Task>("Task", "task-123");
if (task) {
  console.log(task.title);
}
```

**Signature:** `get<T>(modelName: string, id: string): Promise<T | null>`

Returns `null` if the model isn't found in either the identity map or storage.

### getCached

Synchronously reads a model from the identity map only. Doesn't check storage or trigger network requests.

```ts
const task = client.getCached<Task>("Task", "task-123");
// Returns null if not in the identity map
```

**Signature:** `getCached<T>(modelName: string, id: string): T | null`

This is the fastest way to read a model, but it only works for models already loaded into memory.

### ensureModel

Guarantees a model is available locally. If not cached, it loads from storage. If the model's load strategy is `"lazy"`, `"partial"`, or `"explicitlyRequested"`, it fetches from the server via `batchLoad`.

```ts
const task = await client.ensureModel<Task>("Task", "task-123");
```

**Signature:** `ensureModel<T>(modelName: string, id: string): Promise<T | null>`

The loading behavior depends on the model's load strategy:

| Load strategy                                    | Behavior                                                  |
| ------------------------------------------------ | --------------------------------------------------------- |
| `"instant"`                                      | Returns from identity map or storage. No network request. |
| `"lazy"` / `"partial"` / `"explicitlyRequested"` | Fetches from server via `batchLoad` if not cached.        |
| `"local"`                                        | Returns from identity map or storage. No network request. |

### getAll

Returns all models of a given type from the identity map. Supports filtering, sorting, and pagination.

```ts
// All tasks
const tasks = await client.getAll<Task>("Task");

// Filtered and sorted
const activeTasks = await client.getAll<Task>("Task", {
  where: (t) => t.status !== "done",
  orderBy: (a, b) => a.priority - b.priority,
  limit: 20,
});
```

**Signature:** `getAll<T>(modelName: string, options?: QueryOptions<T>): Promise<T[]>`

### query

Queries models with full filtering, sorting, pagination, and metadata.

```ts
const result = await client.query<Task>("Task", {
  where: (t) => t.projectId === "proj-123",
  orderBy: (a, b) => a.createdAt - b.createdAt,
  limit: 10,
  offset: 0,
});

console.log(result.data); // Task[]
console.log(result.hasMore); // boolean
console.log(result.totalCount); // number
```

**Signature:** `query<T>(modelName: string, options?: QueryOptions<T>): Promise<QueryResult<T>>`

See the [Queries](/docs/packages/sync-client/queries) page for detailed query API documentation.

## Creating models

### create

Creates a new model instance. Generates a UUID if no `id` is provided.

```ts
const task = await client.create<Task>("Task", {
  title: "New task",
  status: "todo",
  priority: 0,
  projectId: "proj-123",
});

console.log(task.id); // Auto-generated UUID
```

**Signature:** `create<T>(modelName: string, data: T, options?): Promise<T>`

**Options:**

```ts
await client.create("Task", data, {
  onTransactionCreated: (tx) => {
    console.log("Transaction ID:", tx.clientTxId);
  },
});
```

## Updating models

### update

Updates an existing model with partial changes. Reads the current model from the identity map and computes original values for the changed fields. Throws if the model isn't found in the identity map.

```ts
const updated = await client.update<Task>("Task", "task-123", {
  title: "Updated title",
  priority: 2,
});
```

**Signature:** `update<T>(modelName: string, id: string, changes: Partial<T>, options?): Promise<T>`

**Options:**

```ts
await client.update("Task", "task-123", changes, {
  original: { title: "Previous title" }, // Explicit originals
  onTransactionCreated: (tx) => {
    console.log("Transaction:", tx.clientTxId);
  },
});
```

## Deleting models

### delete

Permanently deletes a model instance. Reads the current model from the identity map and stores the original data in the transaction for undo. Throws if the model isn't found in the identity map.

```ts
await client.delete("Task", "task-123");
```

**Signature:** `delete(modelName: string, id: string, options?): Promise<void>`

## Archiving models

### archive

Soft-deletes a model by setting an `archivedAt` timestamp. Queries exclude archived models by default.

```ts
await client.archive("Task", "task-123");
```

**Signature:** `archive(modelName: string, id: string, options?): Promise<void>`

**Options:**

```ts
await client.archive("Task", "task-123", {
  archivedAt: "2025-01-15T10:30:00.000Z", // Custom timestamp
  original: { archivedAt: undefined },
  onTransactionCreated: (tx) => {
    /* ... */
  },
});
```

### unarchive

Restores a soft-deleted model by clearing the `archivedAt` field.

```ts
await client.unarchive("Task", "task-123");
```

**Signature:** `unarchive(modelName: string, id: string, options?): Promise<void>`

## Other operations

### syncNow

Forces an immediate sync with the server.

```ts
await client.syncNow();
```

### clearAll

Clears all local data including identity maps, storage, pending loads, and undo history.

```ts
await client.clearAll();
```

### getIdentityMap

Returns the raw identity map for a model type. Useful for advanced scenarios.

```ts
const map = client.getIdentityMap<Task>("Task");
for (const [id, task] of map) {
  console.log(id, task.title);
}
```

### isModelMissing

Checks whether the client looked up a model but didn't find it. The React hooks use this internally to avoid repeated failed lookups.

```ts
const missing = client.isModelMissing("Task", "task-123");
```
