`@stratasync/client` is the runtime orchestrator for the Strata Sync engine. It manages the full sync lifecycle: bootstrapping, delta processing, optimistic mutations, outbox persistence, identity maps, query execution, and undo/redo history.

## What it provides

- **createSyncClient** -- factory function that wires up storage, transport, reactivity, and returns a `SyncClient` instance
- **CRUD operations** -- `get`, `getCached`, `ensureModel`, `getAll`, `create`, `update`, `delete`, `archive`, `unarchive`
- **Query engine** -- filter operators (`eq`, `neq`, `gt`, `lt`, `isIn`, `contains`, `matches`), combinators (`and`, `or`, `not`), sorting (`sortBy`, `combineSorts`)
- **Event system** -- subscribe to sync lifecycle events (`syncStart`, `syncComplete`, `syncError`, `stateChange`, `connectionChange`, `modelChange`, `rebaseConflict`)
- **Undo/redo** -- client-side undo and redo with full transaction history
- **Identity map** -- in-memory cache that deduplicates model instances by ID
- **Outbox manager** -- persists pending mutations for offline-first support

## Installation

```bash
npm install @stratasync/client
```

`@stratasync/client` depends on `@stratasync/core` and `@stratasync/yjs` as workspace dependencies. It has no peer dependencies.

## Quick start

```ts
import { createSyncClient } from "@stratasync/client";
import { createIndexedDbStorage } from "@stratasync/storage-idb";
import { createGraphQLTransport } from "@stratasync/transport-graphql";
import { createMobXReactivity } from "@stratasync/mobx";

const client = createSyncClient({
  storage: createIndexedDbStorage(),
  transport: createGraphQLTransport({
    endpoint: "https://api.example.com/graphql",
    syncEndpoint: "https://api.example.com/sync",
    wsEndpoint: "wss://api.example.com/sync",
    auth: { getAccessToken: async () => "your-token" },
  }),
  reactivity: createMobXReactivity(),
  optimistic: true,
  batchMutations: true,
});

// Start the sync lifecycle
await client.start();

// Create a task
const task = await client.create("Task", {
  title: "My task",
  status: "todo",
});

// Query tasks
const { data: tasks } = await client.query("Task", {
  where: (t) => t.status === "todo",
  orderBy: (a, b) => a.priority - b.priority,
});

// Stop when done
await client.stop();
```

## Architecture role

`@stratasync/client` sits between the core primitives and framework bindings:

```
sync-core, sync-yjs
  ^-- sync-client
        ^-- sync-react
              ^-- sync-next
```

The client package is framework-agnostic. It doesn't depend on React, Next.js, or any browser API beyond what the injected adapters provide. The React hooks in `@stratasync/react` are thin wrappers around the `SyncClient` interface.

## SyncClient interface

The `createSyncClient` factory returns an object implementing the `SyncClient` interface:

| Property / Method                   | Type                                                | Description                               |
| ----------------------------------- | --------------------------------------------------- | ----------------------------------------- |
| `state`                             | `SyncClientState`                                   | Current sync state.                       |
| `connectionState`                   | `ConnectionState`                                   | Current connection state.                 |
| `lastSyncId`                        | `number`                                            | Last sync ID received from server.        |
| `lastError`                         | `Error \| null`                                     | Last error (if any).                      |
| `clientId`                          | `string`                                            | Client instance identifier.               |
| `yjs`                               | `{ documentManager, presenceManager } \| undefined` | Yjs managers (if configured).             |
| `start()`                           | `Promise<void>`                                     | Start the sync lifecycle.                 |
| `stop()`                            | `Promise<void>`                                     | Stop the sync lifecycle.                  |
| `get(modelName, id)`                | `Promise<T \| null>`                                | Get a model by ID.                        |
| `getCached(modelName, id)`          | `T \| null`                                         | Get from identity map only.               |
| `ensureModel(modelName, id)`        | `Promise<T \| null>`                                | Load if not cached.                       |
| `getAll(modelName, options?)`       | `Promise<T[]>`                                      | Get all models of a type.                 |
| `query(modelName, options?)`        | `Promise<QueryResult<T>>`                           | Query with filters.                       |
| `create(modelName, data)`           | `Promise<T>`                                        | Create a new model.                       |
| `update(modelName, id, changes)`    | `Promise<T>`                                        | Update a model.                           |
| `delete(modelName, id)`             | `Promise<void>`                                     | Delete a model.                           |
| `archive(modelName, id)`            | `Promise<void>`                                     | Soft-delete a model.                      |
| `unarchive(modelName, id)`          | `Promise<void>`                                     | Restore a model.                          |
| `canUndo()`                         | `boolean`                                           | Whether undo is available.                |
| `canRedo()`                         | `boolean`                                           | Whether redo is available.                |
| `undo()`                            | `Promise<void>`                                     | Undo last operation.                      |
| `redo()`                            | `Promise<void>`                                     | Redo last undone operation.               |
| `onEvent(callback)`                 | `() => void`                                        | Subscribe to events. Returns unsubscribe. |
| `onStateChange(callback)`           | `() => void`                                        | Subscribe to state changes.               |
| `onConnectionStateChange(callback)` | `() => void`                                        | Subscribe to connection changes.          |
| `getPendingCount()`                 | `Promise<number>`                                   | Pending transaction count.                |
| `syncNow()`                         | `Promise<void>`                                     | Force an immediate sync.                  |
| `clearAll()`                        | `Promise<void>`                                     | Clear all local data.                     |
| `getIdentityMap(modelName)`         | `Map<string, T>`                                    | Raw identity map access.                  |
| `isModelMissing(modelName, id)`     | `boolean`                                           | Whether a model wasn't found.             |

## Advanced exports

The following classes and functions are exported for advanced use cases such as custom orchestration, testing, or direct identity map manipulation. Most applications won't need these directly:

| Export                | Description                                                                      |
| --------------------- | -------------------------------------------------------------------------------- |
| `IdentityMap`         | In-memory cache that deduplicates model instances by ID.                         |
| `IdentityMapRegistry` | Manages multiple `IdentityMap` instances, one per model type.                    |
| `OutboxManager`       | Manages the pending transaction outbox for offline-first persistence.            |
| `SyncOrchestrator`    | Coordinates the full sync lifecycle (bootstrap, delta processing, reconnection). |
| `executeQuery`        | Runs a query against the identity map with filter operators and sorting.         |
