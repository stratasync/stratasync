`@stratasync/react` provides React bindings for the Strata Sync engine. It includes a `SyncProvider` context component and a comprehensive set of hooks for reading models, querying data, monitoring connection state, and collaborating in real-time.

## What it provides

- **SyncProvider** -- context provider that wraps your component tree and manages the sync client lifecycle
- **Data hooks** -- `useModel`, `useModelState`, `useModelSuspense`, `useQuery`, `useQueryAll`, `useQueryCount`
- **Connection hooks** -- `useConnectionState`, `useIsOffline`, `usePendingCount`, `useSync`
- **Client hooks** -- `useSyncClient`, `useSyncClientInstance`, `useSyncReady`, `useSyncState`
- **Collaborative hooks** -- `useYjsDocument`, `useYjsPresence`

## Installation

```bash
npm install @stratasync/react
```

## Quick start

### 1. Set up the provider

Wrap your component tree with `SyncProvider` and pass a configured sync client:

```tsx
import { SyncProvider } from "@stratasync/react";
import { createSyncClient } from "@stratasync/client";
import { createIndexedDbStorage } from "@stratasync/storage-idb";
import { createGraphQLTransport } from "@stratasync/transport-graphql";
import { createMobXReactivity } from "@stratasync/mobx";

const client = createSyncClient({
  storage: createIndexedDbStorage(),
  transport: createGraphQLTransport({
    /* ... */
  }),
  reactivity: createMobXReactivity(),
});

function App() {
  return (
    <SyncProvider client={client}>
      <TaskList />
    </SyncProvider>
  );
}
```

### 2. Use hooks in components

```tsx
import { useQuery, useModel } from "@stratasync/react";
import { eq, sortBy } from "@stratasync/client";

function TaskList() {
  const { data: tasks, isLoading } = useQuery<Task>("Task", {
    where: (t) => t.status !== "done",
    orderBy: sortBy("priority", "desc"),
  });

  if (isLoading) {
    return <div>Loading...</div>;
  }

  return (
    <ul>
      {tasks.map((task) => (
        <li key={task.id}>{task.title}</li>
      ))}
    </ul>
  );
}
```

## SyncProvider

The `SyncProvider` component creates a React context that holds the sync client and its reactive state. All sync hooks must be called within this provider.

```tsx
<SyncProvider
  client={syncClient}
  autoStart={true} // Start client on mount (default: true)
  autoStop={true} // Stop client on unmount (default: true)
>
  {children}
</SyncProvider>
```

### SyncProviderProps

| Prop        | Type         | Default  | Description                                    |
| ----------- | ------------ | -------- | ---------------------------------------------- |
| `client`    | `SyncClient` | Required | The sync client instance.                      |
| `children`  | `ReactNode`  | Required | Child components.                              |
| `autoStart` | `boolean`    | `true`   | Automatically call `client.start()` on mount.  |
| `autoStop`  | `boolean`    | `true`   | Automatically call `client.stop()` on unmount. |

### SyncContext

The raw React context object is also exported as `SyncContext`. This is primarily useful for testing, where you may want to provide a mock sync client directly:

```tsx
import { SyncContext } from "@stratasync/react";

// In tests:
<SyncContext.Provider value={mockContextValue}>
  <ComponentUnderTest />
</SyncContext.Provider>;
```

### Context value

The provider exposes the following values via context (accessible through `useSyncClient`):

| Field             | Type              | Description                                            |
| ----------------- | ----------------- | ------------------------------------------------------ |
| `client`          | `SyncClient`      | The sync client instance.                              |
| `state`           | `SyncClientState` | Current sync state.                                    |
| `connectionState` | `ConnectionState` | Current connection state.                              |
| `lastSyncId`      | `number`          | Last sync ID from server.                              |
| `backlog`         | `number`          | Pending transaction count.                             |
| `error`           | `Error \| null`   | Last sync error.                                       |
| `isReady`         | `boolean`         | `true` when state is `"syncing"`.                      |
| `isSyncing`       | `boolean`         | `true` when state is `"syncing"` or `"bootstrapping"`. |
| `isOffline`       | `boolean`         | `true` when connection state is `"disconnected"`.      |
| `clientId`        | `string`          | Client instance identifier.                            |
