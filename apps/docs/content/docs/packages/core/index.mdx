`@stratasync/core` is the foundation of the Strata Sync stack. Every other package depends on it for model definitions, schema metadata, transaction formats, and delta processing.

## Installation

Install the package with npm.

```bash
npm install @stratasync/core
```

`@stratasync/core` has no peer dependencies. It bundles `mobx` as a regular dependency for its internal observable property system.

## Package exports

You can import all public APIs (application programming interfaces) from the package entry point. The most commonly used exports are grouped below by category.

```ts
import {
  // Models
  Model,
  Collection,

  // Decorators
  ClientModel,
  Property,
  Reference,
  ManyToOne,
  OneToMany,
  BackReference,

  // Schema
  ModelRegistry,
  computeSchemaHash,

  // Transactions
  createInsertTransaction,
  createUpdateTransaction,
  createDeleteTransaction,
  createUndoTransaction,
  serializeTransaction,
  deserializeTransaction,

  // Sync
  applyDeltas,
  createBatchedApplier,
  rebaseTransactions,

  // Utils
  generateUUID,
  generateModelId,
  getOrCreateClientId,
} from "@stratasync/core";
```

Additional exports for serialization helpers (`serializeBatch`, `batchToJson`), ephemeral properties (`EphemeralProperty`), reference collections (`ReferenceArray`, `ReferenceCollection`), and internal utilities are available from the same entry point.

## Architecture role

`@stratasync/core` sits at the bottom of the dependency graph. It defines the data model, schema, and transaction format that all higher-level packages consume.

```
core
  ^-- y-doc
  ^-- client
        ^-- react
              ^-- next
```

The package is framework-agnostic with no React, Next.js, or browser-specific dependencies. You can use it in Node.js, Deno, or any JavaScript runtime that supports ES modules (ESM).
