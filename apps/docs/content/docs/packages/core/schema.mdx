The schema system in `@stratasync/core` provides runtime metadata about your models, their properties, and relationships. The `ModelRegistry` is the central registry that decorators write to and the sync engine reads from.

## ModelRegistry

`ModelRegistry` serves two roles:

1. **Static registry** -- class-level maps populated by decorators at module load time
2. **Instance registry** -- constructed from a schema snapshot for server-side or serialized schema usage

### Static API

The static methods operate on the global registry populated by decorators:

```ts
import { ModelRegistry } from "@stratasync/core";

// Check registered models
const names = ModelRegistry.getModelNames();
// ["Workspace", "Project", "Task", ...]

// Get metadata for a model
const meta = ModelRegistry.getModelMetadata("Task");
// { name: "Task", loadStrategy: "instant" }

// Get property metadata
const props = ModelRegistry.getModelProperties("Task");
// Map { "title" => { type: "property" }, "projectId" => { type: "reference", ... } }

// Get bootstrap model names (instant load strategy)
const bootstrapModels = ModelRegistry.getBootstrapModelNames();
// ["Workspace", "Project", "Task"]

// Get partial model names
const partialModels = ModelRegistry.getPartialModelNames();
// ["Comment", "Attachment"]

// Compute schema hash
const hash = ModelRegistry.getSchemaHash();
// "a1b2c3d4"
```

### Static method reference

| Method                                 | Return type                     | Description                                   |
| -------------------------------------- | ------------------------------- | --------------------------------------------- |
| `registerModel(name, ctor, meta)`      | `void`                          | Registers a model (called by `@ClientModel`). |
| `registerProperty(target, name, meta)` | `void`                          | Registers a property (called by decorators).  |
| `getModelNames()`                      | `string[]`                      | All registered model names.                   |
| `getModelMetadata(name)`               | `ModelMetadata \| undefined`    | Metadata for a specific model.                |
| `getModelConstructor(name)`            | `ModelConstructor \| undefined` | Constructor class for a model name.           |
| `getModelName(ctor)`                   | `string \| undefined`           | Model name for a constructor.                 |
| `hasModel(name)`                       | `boolean`                       | Whether a model is registered.                |
| `getModelMetadataEntries()`            | `[string, ModelMetadata][]`     | All model metadata entries.                   |
| `getBootstrapModelNames()`             | `string[]`                      | Models with `loadStrategy: "instant"`.        |
| `getPartialModelNames()`               | `string[]`                      | Models with `loadStrategy: "partial"`.        |
| `getModelProperties(name)`             | `Map<string, PropertyMetadata>` | All property metadata for a model.            |
| `getReferencedProperties(name)`        | `Map<string, PropertyMetadata>` | Reference/collection properties.              |
| `getPropertyNames(name)`               | `string[]`                      | Sorted property names.                        |
| `getSchemaHash()`                      | `string`                        | Deterministic hash of the full schema.        |
| `snapshot()`                           | `ModelRegistrySnapshot`         | Serializable snapshot of the registry.        |

### Instance API

For server-side usage or when working with serialized schemas, construct a `ModelRegistry` instance:

```ts
import { ModelRegistry } from "@stratasync/core";

// From a SchemaDefinition
const registry = new ModelRegistry({
  models: {
    Task: {
      loadStrategy: "instant",
      fields: {
        id: {},
        title: { type: "string" },
        status: { type: "string" },
      },
      relations: {
        project: {
          kind: "belongsTo",
          model: "Project",
          foreignKey: "projectId",
        },
      },
    },
  },
});

// Instance methods mirror static ones
const names = registry.getModelNames();
const hash = registry.getSchemaHash();
const model = registry.getModel("Task");
const bootstrapModels = registry.getBootstrapModels();
```

| `getPartialModelNames()`    | `string[]`                      | Models with `partial` load strategy.    |

## computeSchemaHash

Computes a deterministic hash of the schema for version checking. The hash is used to detect schema mismatches between the client and server.

```ts
import { computeSchemaHash, ModelRegistry } from "@stratasync/core";

// From the global registry
const hash = computeSchemaHash(ModelRegistry.snapshot());

// From a schema definition
const hash2 = computeSchemaHash({
  models: {
    Task: {
      meta: { name: "Task", loadStrategy: "instant" },
      properties: {
        title: { type: "property" },
        status: { type: "property" },
      },
    },
  },
});
```

**Signature:** `computeSchemaHash(input: ModelRegistrySnapshot | SchemaDefinition): string`

The hash algorithm:

1. Canonicalizes the snapshot (sorts model names, property names, removes `undefined` values)
2. Serializes to JSON
3. Applies a djb2 hash variant
4. Returns an 8-character hexadecimal string

The sync client compares schema hashes during bootstrap to detect when the client schema has changed and local storage needs to be cleared and re-bootstrapped.

## Load strategies

Load strategies determine how and when model data is fetched from the server. Set the strategy in the `@ClientModel` decorator:

```ts
@ClientModel("Project", { loadStrategy: "instant" })
class Project extends Model {
  /* ... */
}
```

### Strategy details

Each load strategy controls bootstrap inclusion, on-demand fetching, and caching behavior:

| Strategy                | Bootstrap | On-demand               | Cache             | Use case                           |
| ----------------------- | --------- | ----------------------- | ----------------- | ---------------------------------- |
| `"instant"`             | Included  | --                      | Always in memory  | Core data needed on every page.    |
| `"lazy"`                | Excluded  | Fetched on first access | Cached after load | Referenced data not always needed. |
| `"partial"`             | Excluded  | Fetched by index        | Cached by index   | Large collections loaded by FK.    |
| `"explicitlyRequested"` | Excluded  | Only via `ensureModel`  | Cached after load | Rarely needed data.                |
| `"local"`               | Excluded  | Never synced            | Local only        | Device-specific settings.          |

### Partial load mode

For models with `loadStrategy: "partial"`, you can set a hydration priority:

```ts
@ClientModel("Comment", {
  loadStrategy: "partial",
  partialLoadMode: "regular",
})
class Comment extends Model {
  /* ... */
}
```

| Mode            | Description                                     |
| --------------- | ----------------------------------------------- |
| `"full"`        | Highest priority, hydrated as soon as possible. |
| `"regular"`     | Normal priority.                                |
| `"lowPriority"` | Lowest priority, hydrated last.                 |

## Schema types

### ModelMetadata

```ts
interface ModelMetadata {
  name: string;
  loadStrategy: LoadStrategy;
  partialLoadMode?: PartialLoadMode;
  usedForPartialIndexes?: boolean;
  schemaVersion?: number;
  tableName?: string;
}
```

### PropertyMetadata

```ts
interface PropertyMetadata {
  type: PropertyType;
  lazy?: boolean;
  serializer?: PropertySerializer<unknown>;
  indexed?: boolean;
  nullable?: boolean;
  referenceModel?: string;
  inverseProperty?: string;
  foreignKey?: string;
  through?: string;
}
```

### PropertyType

```ts
type PropertyType =
  | "property"
  | "ephemeralProperty"
  | "reference"
  | "referenceModel"
  | "referenceCollection"
  | "backReference"
  | "referenceArray";
```

### SchemaDefinition

The schema definition format is used for programmatic schema construction (as opposed to decorator-based):

```ts
interface SchemaDefinition {
  models: Record<string, ModelDefinition>;
}

interface ModelDefinition {
  name?: string;
  loadStrategy?: LoadStrategy;
  partialLoadMode?: PartialLoadMode;
  primaryKey?: string;
  groupKey?: string;
  fields?: Record<string, FieldDefinition>;
  relations?: Record<string, RelationDefinition>;
  indexes?: ModelIndexDefinition[];
  schemaVersion?: number;
  tableName?: string;
}
```
