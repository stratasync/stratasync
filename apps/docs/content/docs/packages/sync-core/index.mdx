`@stratasync/core` is the foundation of the Strata Sync stack. It provides the model base class, decorator-based schema registration, transaction creation and serialization, delta processing, and the schema registry. Every other package in the stack depends on it.

## What it provides

The core package includes the following primitives for building sync-enabled applications:

- **Model base class** -- extend `Model` to define synced entities with change tracking, hydration, and serialization
- **Decorators** -- `@ClientModel`, `@Property`, `@EphemeralProperty`, `@Reference`, `@ManyToOne`, `@OneToMany`, `@BackReference`, `@ReferenceArray`, `@ReferenceCollection` for declarative schema definitions
- **Transactions** -- factory functions for creating insert, update, delete, archive, unarchive, and undo transactions
- **Schema registry** -- `ModelRegistry` for runtime model metadata, `computeSchemaHash` for deterministic schema versioning
- **Delta processing** -- `applyDeltas`, `createBatchedApplier`, `rebaseTransactions` for server-sequenced conflict resolution
- **Reactivity adapter** -- pluggable abstraction for observable state (MobX, signals, and others)
- **ID utilities** -- UUID generation, idempotency keys, client ID management

## Installation

```bash
npm install @stratasync/core
```

`@stratasync/core` has no peer dependencies. It bundles `mobx` as a regular dependency for its internal observable property system.

## Architecture role

`@stratasync/core` sits at the bottom of the dependency graph. It defines the data model, schema, and transaction format that all other packages consume:

```
sync-core
  ^-- sync-yjs
  ^-- sync-client
        ^-- sync-react
              ^-- sync-next
```

The core package is framework-agnostic and has no React, Next.js, or browser-specific dependencies. It can be used in Node.js, Deno, or any JavaScript runtime that supports ES modules.
