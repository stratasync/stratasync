description: MobX reactivity adapter for observable model properties and automatic UI updates in Strata Sync.

`@stratasync/mobx` implements the `ReactivityAdapter` interface from `@stratasync/core` using MobX. It makes synced model properties observable so MobX-powered React components re-render when data changes.

## What it provides

This package exports the adapter, a factory function, and re-exported core APIs.

- **`mobxReactivityAdapter`** -- a pre-configured singleton implementing `ReactivityAdapter` with MobX observables, computed values, reactions, and batched updates
- **`createMobXReactivity`** -- a factory function returning the same adapter (useful for explicit instantiation)
- **Core re-exports** -- model decorators and base classes from `@stratasync/core` (see [Re-exported core APIs](#re-exported-core-apis))

## Installation

```bash
npm install @stratasync/mobx mobx
```

### Peer dependencies

| Package | Version  |
| ------- | -------- |
| `mobx`  | `^6.0.0` |

## Usage

Pass the adapter when creating the sync client.

```ts
import { createSyncClient } from "@stratasync/client";
import { mobxReactivityAdapter } from "@stratasync/mobx";

const client = createSyncClient({
  reactivity: mobxReactivityAdapter,
  // ... storage, transport
});
```

With this adapter installed, all model properties decorated with `@Property`, `@Reference`, `@ReferenceArray`, and others become MobX observables.

- MobX `autorun`, `reaction`, and `when` track reads and re-execute on changes
- React components wrapped with `observer()` from `mobx-react-lite` re-render when observed properties change
- Batch updates from the sync engine run inside `runInAction` to avoid unnecessary intermediate renders

## API reference

### mobxReactivityAdapter

A singleton object implementing `ReactivityAdapter` from `@stratasync/core`.

```ts
import { mobxReactivityAdapter } from "@stratasync/mobx";
```

The adapter implements these methods:

| Method                            | MobX Implementation               | Description                                                      |
| --------------------------------- | --------------------------------- | ---------------------------------------------------------------- |
| `createBox(initialValue)`         | `observable` + `makeObservable`   | Creates a reactive boxed value.                                  |
| `createMap(entries)`              | `observable.map`                  | Creates a reactive key-value map.                                |
| `createArray(items)`              | `observable.array`                | Creates a reactive array.                                        |
| `makeObservable(target)`          | `observable(target)`              | Makes a plain object deeply observable.                          |
| `batch(fn)`                       | `runInAction(fn)`                 | Batches multiple updates into a single reaction cycle.           |
| `runInAction(fn)`                 | `runInAction(fn)`                 | Runs a function inside a MobX action.                            |
| `reaction(expr, effect, options)` | `reaction(expr, effect, options)` | Creates a reaction that tracks an expression and runs an effect. |
| `computed(getter)`                | `computed(getter)`                | Creates a MobX computed value.                                   |

### createMobXReactivity

A factory function that returns the `mobxReactivityAdapter` singleton. Both approaches are equivalent.

```ts
import { createMobXReactivity, mobxReactivityAdapter } from "@stratasync/mobx";

// These are the same object
const adapter1 = mobxReactivityAdapter;
const adapter2 = createMobXReactivity();
// adapter1 === adapter2
```

**Signature:** `createMobXReactivity(): ReactivityAdapter`

### ReactivityAdapter interface

The adapter conforms to this interface from `@stratasync/core`.

```ts
interface ReactivityAdapter {
  createBox<T>(initialValue: T, options?: ObservableOptions): ObservableBox<T>;
  createMap<K, V>(
    entries?: Iterable<[K, V]>,
    options?: ObservableOptions
  ): ObservableMap<K, V>;
  createArray<T>(items?: T[], options?: ObservableOptions): ObservableArray<T>;
  makeObservable<T extends object>(target: T, options?: ObservableOptions): T;
  batch(fn: () => void): void;
  runInAction<T>(fn: () => T): T;
  reaction<T>(
    expression: () => T,
    effect: (value: T) => void,
    options?: ReactionOptions
  ): DisposeFn;
  computed<T>(getter: () => T, options?: ObservableOptions): { get(): T };
}
```

## Re-exported core APIs

For convenience, you can import model base classes and decorators directly from `@stratasync/mobx` instead of `@stratasync/core`.

```ts
import {
  Model,
  Collection,
  ClientModel,
  Property,
  Reference,
  ReferenceArray,
  mobxReactivityAdapter,
} from "@stratasync/mobx";
```

This lets you define models and configure reactivity from a single import.

```ts
import {
  Model,
  ClientModel,
  Property,
  mobxReactivityAdapter,
} from "@stratasync/mobx";

@ClientModel("Task")
class Task extends Model {
  @Property() title!: string;
  @Property() description!: string;
  @Property() priority!: number;
}
```

## How it works

The adapter wraps all model property storage in MobX observables, so any `autorun`, `reaction`, `computed`, or `observer` component that reads a property receives notification on change. Incoming sync updates (deltas, bootstrap data) are wrapped in `runInAction` to batch multiple field updates into a single reaction cycle.

## Architecture role

`@stratasync/mobx` bridges `@stratasync/core` with MobX.

```
sync-core (defines ReactivityAdapter interface)
  ^-- sync-mobx (implements ReactivityAdapter with MobX)
        ^-- sync-client (uses adapter for observable state)
```

The sync system is reactivity-agnostic. The same `ReactivityAdapter` interface can be implemented with other state management libraries.
