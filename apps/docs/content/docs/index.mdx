Build instant, offline-capable apps that sync. Strata Sync is a local-first, server-sequenced sync engine for TypeScript, React, and Next.js. Every read comes from a local store, so the UI stays fast. Every write enters an optimistic outbox, so mutations feel instant. A monotonic sync log on the server ensures all clients converge to the same state.

Strata Sync implements the server-sequenced sync architecture that [Linear](https://linear.app) pioneered and described in their engineering blog posts and conference talks. Strata Sync extends it with Yjs CRDT collaboration, built-in undo/redo, pluggable adapters, and framework integrations.

## Key features

Strata Sync provides a focused set of capabilities for building collaborative, offline-ready applications.

- **Local-first reads** -- render from IndexedDB with zero network round-trips
- **Server-sequenced consistency** -- a monotonic `syncId` orders all changes globally
- **MobX reactivity** -- field changes re-render only the observing components
- **Offline writes** -- a persistent outbox replays mutations on reconnect
- **Rich-text collaboration** -- Yjs CRDT (Conflict-free Replicated Data Type) integration for multi-user editing
- **Undo/redo** -- transaction-based history tracking
- **Field-level conflict resolution** -- LWW (Last-Writer-Wins) rebase with configurable strategies
- **Partial replication** -- load strategies control eager vs. on-demand sync
- **Type-safe schema** -- decorators define models; typed hooks bind them to React

## How it works

Strata Sync follows a five-step workflow from schema to sync.

1. Define models with decorators (`@ClientModel`, `@Property`, `@ManyToOne`, `@OneToMany`).
2. Create a sync client by composing storage, transport, and reactivity adapters.
3. Wrap your React tree with `SyncProvider` to manage the client lifecycle.
4. Read data with hooks (`useModel`, `useQuery`) that return reactive, auto-updating results.
5. Write data through `create`, `update`, and `delete` -- applied optimistically and synced in the background.
